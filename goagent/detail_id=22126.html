虽然我部署成功了，其中碰到一些问题，补充说明下，方便碰到类似问题的人参考

1. 在 heroku login 登录成功后，提示 updating Heroku CLI...
这是软件在自动更新，只有等它更新完，然后接着以下步骤

2. git add . -A
提示 warning: LF will be replaced by CRLF in ...
一开始我是手动去修改换行编码的，后来发现不需要这么做，只需要再执行一遍相同的命令，软件会自动改编码

3. git commit -am "make it better"
提示错误信息（啥信息没记下来...）时，需要告诉软件你是谁..

git config --global user.email "user@user.com"
把以上 user@user.com 替换成你的登录邮箱

然后再执行 git commit -am "make it better"

4. git push heroku master
执行这步如果碰到错误：

fatal: 'heroku' does not appear to be a git repository
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

（注：一般不会碰到这个错误信息，我也不太清楚具体原因，不过解决方法是不会错的）

则先 heroku git:remote -a 所建的app名称

再 git push heroku master 即可
