#1 linhaico...@gmail.com 
设置蓝灯为goagent的前置代理，干嘛搞这么复杂，直接在goagent里的配置文件proxy.ini里设置就解决了。

[proxy]
enable = 1
autodetect = 1
host = 127.0.0.1
port = 8787

#3 linhaico...@gmail.com 
再给你讲个meow(蓝灯前置，goagent后置)
打开meow文件夹下的rule.txt文件
添加以下这句: (其实人家打#号注释那段已经讲得很明白了。。)
proxy = http://127.0.0.1:8787

然后修改goagent的proxy.ini文件
[proxy]
enable = 1
autodetect = 1
host = 127.0.0.1
port = 4411

然后依次打开蓝灯、meow、goagent。

browser <-> meow <-> ss server <-> youtube
呢？

答案是，完全可以啊！


那goaent + meow有什么好处呢？
有些人是为了goagent里的autorange功能，更多的似乎只是在“跟着潮流走” XD

autorange有什么用？
分段下载，从而利用多线程
单线程就是从头到尾，按部就班；分段启用多线程可以＊同时＊对一个文件的不同部分下载，有个加速的作用（要看具体情况，并不绝对）

结论：
用了meow又想要autorange，就加上goagent
不需要，就单用meow（或任何其它的ss客户端）


附１：
meow通过ss翻墙，goagent通过gae，两个是完全不同的独立的软件
所谓“meow让goagent复活了”之类的说法，偶只能说：我特的法可？！

附２：
autorange最常见的应用多数就是youtube，firefox里关于视频的设置已经有分段的选项了
about:config?filter=media.fragmented

附３：
proxy chain不涉及技术也该能想到一个问题，即速度永远取决于最慢的那一环！
