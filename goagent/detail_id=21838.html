#12 warmingf...@gmail.com 
#8 3proxy 很牛逼，代理 / 端口转发用的。 "It includes HTTP proxy with HTTPS and FTP support, SOCKSv4/SOCKSv4.5/SOCKSv5 proxy (socks/socks.exe), POP3 proxy, SMTP proxy, AIM/ICQ proxy (icqpr/icqpr.exe), MSN messenger / Live messenger proxy (msnpr/msnpr.exe), FTP proxy, caching DNS proxy, TCP and UDP portmappers. " 同类软件中从来没见过这么多功能的。

例如出现一种情况：

你使用蓝灯，它的访问 107.191.44.7 ，而这在你那非常慢。这时你会想到换一个离你的 CloudFlare IP 地址（ 蓝灯支持使用 CloudFlare CDN 加速），如香港 / 台湾的 CloudFlare 节点。因此你想到了重定向，基于域名访问的地址可以通过修改 hosts 或本地 DNS 解析以解析到另一个 IP 地址而非常轻松地实现重定向，但是蓝灯访问的不是域名，而是 IP 地址，此时 3proxy 可解决此困局——编写规则使蓝灯访问 107.191.44.7 的流量重定向到另一个 CloudFlare IP 地址 211.11.31.38 （举例）以提高速度。 

在 bin 文件夹下新建 config.txt 内容：

auth iponly

#本规则要处理蓝灯访问的流量（此处应该是一个 CloudFlare IP 地址，可通过 Proxifier 创建规则是蓝灯只能访问 107.191.44.7:443 ）
allow 107.191.44.7 443
# TCP portmappers： 使上述流量转而经由 211.11.31.38:443
parent 1000 tcp 211.11.31.38 443
# 为本规则创建本地 socks5 代理以供使用，端口 1380 （同理，可同时在下面创建 http, pop3p 等很多代理类型）
socks -p1380 -i127.0.0.1
# 为本规则创建本地 HTTP 代理以供使用，与 socks5 代理共存（可选）
proxy -p8187 -i127.0.0.1
