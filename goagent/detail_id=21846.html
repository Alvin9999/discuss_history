Issue 21846: CDN加速成为高速翻墙的新手段，原理是否应该给大家普及一下。

Reported by kingtope...@gmail.com, Jul 12, 2015 
最近两天看到有高手分享CDN加速高速分享手段，我测试了一下确实有效果。

但是我觉得不能这样云里雾里、糊里糊涂的用，还是得把原理搞清楚。

本人看了warmingf的教程，仍然不是太明白，有以下几个问题不是太清楚：

1. proxifier在这个CDN加速里面充当了什么角色？为什么一定要把赛风的强制定向到yryirjrhqffsxwpg-a.akamaihd.net? 
host里面不是已经对yryirjrhqffsxwpg-a.akamaihd.net做了DNS解析的定位了吗？

2. 赛风新加坡节点跟akamai的IP具体是什么关系？ 
如果是akamai对赛风新加坡节点进行了CDN加速，具体原理是什么？ 是akamai的IP做了赛风新加坡的前端代理了吗? 这个是不是proxifier起到的作用?



3. *.akamaihd.net这个域名的*的部分是不是可以随便写，还是这个域名是自己申请到的？比如yryirjrhqffsxwpg-a.akamaihd.net这个域名是怎么来的？
yryirjrhqffsxwpg-a 是可以随便写的吗

4. 如何扫描akamai IP？


关于蓝灯：蓝灯的IP跟cloudflare ip是什么关系？ 如果蓝灯IP重定向到cloudflare ip，那么蓝灯还能找到原始的蓝灯节点IP吗？ 其实也就是说cloud IP到底是用什么原理给蓝灯ip加速的？

但是warmingf前面又说赛风是靠akamai域名对赛风加速的，蓝灯是靠cloudflare IP对蓝灯IP加速的。 

我觉得真的有点乱，有点糊涂，需要有人把赛风和蓝灯的CDN加速的原理很清晰的梳理一下。

只有把上面CDN加速的原理搞的很清楚，我们在使用的时候才不会很迷茫。

#1 wdlt...@gmail.com 
1. Proxifier是阻止赛风访问到其他的节点，因为其他节点有其他的域名和IP地址，Hosts是无法强制指定IP地址的。
2. Akamai是一个CDN服务，就是前端加速服务，很多大公司使用Akamai的加速服务，包括苹果、微软等就像国内的各种网站云加速，做一个前端的通道。
3. *当然不能随便写，子域名本身就是用于识别是哪个服务的，比如Steam用的是steamstore-a.akamaihd.net、steamcdn-a.akamaihd.net等，HumbleBundle用的是humblebundle-a.akamaihd.net。这个子域名应该是谁申请的，给大家用的话肯定会产生不少费用，不知道能不能承受得住。听说有个做Google镜像的被攻击，一下就欠费几千美元。
4. 其实没有必要扫描IP，Akamai几乎在全球每个地区都有服务器，是按区域解析的，用香港运营商的DNS解析到的就是香港的，而且有很多不同的运营商结果，你只需要找DNS，然后用那个查看解析结果就够了，也可以使用那些批量的工具。
