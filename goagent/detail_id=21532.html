Reported by kingtope...@gmail.com, Jun 15, 2015 
好久没用命令行的Tor，今天用SS做前置meek连接成功，好激动！


因为图形界面的Tor Browser从来就没有正常过，老是报错： “Tor Brower已停止工作，出现一个问题，导致程序停止正常工作”

所以被迫用命令行的Tor,但是obfs3的网桥实在太慢，还动不动连不上。

只有想办法用前置代理，前置代理用了Sock5Proxy 127.0.0.1:1080,从未成功过，用meek的时候老报错：Managed proxy failed to configure the pluggable transport's outgoing proxy. (don't understand proxy URL scheme "socks5")

今天突发奇想：把Sock5Proxy 127.0.0.1:1080改成了 HTTPS5Proxy 127.0.0.1:1080，果然成功了！
好鸡动！ 匿名没有成功，但是鸡动了！

顺便分享一下torrc文件，我瞎JB搞的，能用就行:

HTTPSProxy 127.0.0.1:1080
SocksListenAddress 127.0.0.1

ClientOnly 1
DataDirectory ./Data
GeoIPFile ./Data/geoip
GeoIPv6File ./Data/geoip6
Log notice file ./tor.log

UseBridges 1



ClientTransportPlugin meek exec ./PluggableTransports/meek-client --log ./meek-client.log

#24 kingtope...@gmail.com 
meek部署GAE的成功经验：

1.必须用VPN,否则无法进行google账户认证,推荐使用VPNGate.
2. python的环境变量要设置，pyOpenssl要用pip更新。
3. 要安装go_appengine_sdk_windows（googlecloudSDK可不用安装），并设置go_appengine的环境变量
4. app.yaml的第一行application:后面除了要写appid以外，meek-reflect一定要删除，否则部署会报错（appcfg.py错误）。
5. readme文件不能放在appengine目录里面，appengine目录只能有两个文件：app.yaml和reflect.go
6.最后运行：goapp deploy.
