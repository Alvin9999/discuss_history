这里成为一个自由讨论的社区，各种信息的想法的交流在这进行。
Google Code不久就要关闭了，我们要找一个地方迁移过去，而且要获得大家的认同。

目前比较合适的是Google Group。

其他方案：
+ P2P的平台，虽然更加自由，但是门槛高，没有装就无法访问，稳定性也不好保证。
+ 个人搭论坛的方式，承受攻击的能力太差。
+ Github的方式，好处是不需要翻墙
  感觉感觉不像一个讨论区。


还有没其他方案，大家都来出主意。

我已经试过写脚本过导出Issues里的讨论列表，每个主题下的内容导出，应该也不难。
在测试导入到Google Group时，需要一个Google Admin帐号，谁能帮忙搞一个？

把以前的数据导入进入Google Group，是非常有意义的。
希望大家能够一起来协调合作，维护一个交流的社区。

#34 xxnet....@gmail.com 

1. p2p 的平台，可以作为一个补充
  很多人是来找翻墙的方案，也许只是通过轮子的工具上了google，想寻找一个地方交流。
  让他先去下载、运行一个p2p的软件，对他来说也许这样的工具有很多个，选对了工具、真正运行起来，都不容易。

  而且目前各种p2p 都不够成熟，也容易存在漏洞，被攻击的话，针对性很强。

  简单的说，p2p 是未来的方向，但门槛高、不成熟，目前还不适合作为大众的平台。


  

2. 目前有两个比较好选择：
  Google Group
  Github


  Google Group 
   优势：有群公告、置顶帖子

  Github
   优势：目前不必翻墙

   缺点：
     + 不像论坛
     + 缺乏群公告、置顶帖子。
     + 可以通过在项目首页和Wiki中去达到类似目的。

搭建在appengine上，可能存在的问题：
+ 免费的GAE帐号流量有限，很容易被攻击导致拒绝服务
+ 管理员可以看到用户的ip地址，如果某个无毛渗透进去，会对大家不利
+ 维护成本： 如果发现漏洞，需要升级服务端，而这个权限只有一个google帐号掌握。
  如果创建帐号的人不在了，就无法维护。

相比之下，Google Group：
+ 流量无限，不会被DOS
+ ip地址保密，管理员也无法获取
+ 有漏洞Google会维护
