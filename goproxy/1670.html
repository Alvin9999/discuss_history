ccwufu commented Apr 23, 2017 • edited

360 / 奇虎 C301 : AR9334 , 是big endian 的。
但由于openwrt / lede 默认没有enable FPU emulator ,
导致illegal instruction , (很容易让人误认为是 endian 问题）
(./goproxy: line 2: syntax error: unexpected word (expecting ")") :根据论坛里的例子，　这个错误，非常大的可能是endian 不匹配的问题）

可以自定义编译 openwrt/lede,
OpenWrt:
make kernel_menuconfig
--> Kernel type
---->set MIPS FPU Emulator = y
Build image.

在AR9331上试了一下goproxy 1383 mips (GL.inet 1664 : 16M flahs / 64M RAM / 400MHz CPU, goproxy直接放在/tmp目录下运行）
可以运行， CPU 经常在>~95%
运行一段时间（几分钟）就自动退出，
所以，在低端无线路由器上，　几乎没有实用性

[更新]
把 goproxy 放到/root 目录下运行，
还是比较稳定，
（可能是 goproxy 放在 /tmp 目录下， tmpfs 占用了 内存， 导致 内存不足， 不稳定 )

[更新2]
由于 /root目录是 overlay + squashfs 文件系统， 写操作，非常慢
修改 goproxy.sh
a) goproxy.log 放到 /tmp/goproxy.log
b) mkdir -p /tmp/cache
把 cache 放到 /tmp 目录下 （运行过程中会写cache 目录）
ln -s /tmp/cache /root/goproxy/cache

内存使用， 文件访问 优化 都无法达到满意/稳定的结果 。
应该是 CPU太弱(并且没有硬件FPU, 而是软件模拟FPU) , 经常 ~95%-100%， 导致 路由器重启 。

可能C301: 16M flash , 128M RAM , 560M CPU 可能会好一点。

======================================
OpenWRT running on a AR9331 SoC big endian without FPU

golang/go#18880
Is kernel FPU emulation enabled on the target machine? That is required for running go1.8 on mipsle:

Go now supports 32-bit MIPS on Linux for both big-endian (linux/mips) and little-endian machines (linux/mipsle) that implement the MIPS32r1 instruction set with FPU or kernel FPU emulation. Note that many common MIPS-based routers lack an FPU and have firmware that doesn't enable kernel FPU emulation; Go won't run on such machines.
