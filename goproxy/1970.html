**关键字：废物利用 移动的 goproxy **

理由：安卓 5.x 上可以用 termux ， 4.x 的不能用 termux 。所以。。。。开始折腾。。。。

方案：
使用 busybox + Linux Deploy 组合。这个方案可扩展性强。
除了 goproxy ，以后还可以安装 dns 、http server 等等。这就是一个随身携带的小型服务器。

前提条件：
手机系统 安卓 4.x ，且 root 。（怎么 root ，自行 google 。）

手机配置：

OS: Android 4.3
CPU: 高通 双核
RAM: 512MB
ROM：2GB 可用容量。不用 SD 卡，直接装在内部储存卡上。

使用的 apk ：
busybox 版本：1.27.1
Linux Deploy 版本：v2.0.2
JuiceSSH 版本：v2.1.4

安装顺序： busybox --> Linux Deploy --> JuiceSSH

busybox 和 JuiceSSH 的安装过程不写了。默认就行。不懂自行 google :)
JuiceSSH 在手机上用不多。
重点写下 Linux Deploy 设置、配置和安装过程。可能会没有图。

Linux Deploy 设置 （左上角的"三道杠"）

锁定屏幕：取消勾选
销定Wi-Fi：勾选
wake lock：勾选
语言：简体中文。(你英语好就用英文) 点击左上角的"三道杠"，Setting --> language --> 简体中文。
其它默认

Linux Deploy 属性（部署配置 --- 左下角下载图标）

Containerization method： chroot
发行版： Debian
发行版版本： stretch
架构： armhf （默认。软件自动选择。）
镜像地址： http://mirrors.ustc.edu.cn/debian/
安装类型：文件  （这个是重点。“文件”类型占用空间小。）
安装路径：默认。（${EXTERNAL_STORAGE}/linux.img）
镜像大小：  2048 MB
文件系统：默认。（我选择 ext4）
用户名：xxxx ( ssh 连接用到)
用户密码：xxxx ( ssh 连接用到)
本地化：zh_CN.UTF-8
选择组件： 只保留 SSH 服务器 ( SSH --> Enable -- 勾选 )

其它默认
图形界面： 取消勾选（破手机就不上这个了。。。）
自定义挂载（MOUNTS）： 取消勾选

安装过程注意事项：

  1. 修改"属性"后须要重新"点击"一次"配置"（重新配置）。要点击“停止”才可以“安装”。
  2. 安装过程中如有出错，须“停止”再重新“安装”。
  3. 安装源尽量用国内源。国外源有可能都连不上，且中途容易出错。

安装过程：
右上角选择“安装”开始安装。（显示 <<< deploy 表示完成）
安装完成后“启动(start)”开始运行。（显示 <<< start 表示完成）

如下图：安装完成后，基本系统只占用 314M ，还有 1.6G 的空间。

编译（可选）：
我使用的是自编译。关键字：GOARM=7
如不能自编译，armhf 架构选择的版本是 goproxy_linux_arm-rxxxx

SSH 上传 goproxy ：
使用 ssh 来上传文件。帐号密码是配置时所设定的。
windows 下我用 FileZilla （sftp）来上传文件。简单快捷。

FileZilla 设置：关键是红线部分。

上传前准备工作：

  1. 解压 goproxy_linux_arm-rxxxx.tar.bz2
  2. 拷贝 PC 上 goproxy 目录下的以下文件到上面解压目录下。
cache 目录以及子目录下所有文件
GoProxy.key
GoProxy.crt
*.user.json
proxy.pac

这样做其中一个重要目的：保持 Android 和 PC 端“证书”一样。

新建一个目录，用 FileZilla （sftp）上传到此目录下。结果图：

运行与使用：

# 增加运行权限
chmod +x goproxy

# 后台运行
nohup ./goproxy &>/dev/null &

# 下面这两个是查找和关闭 ---- 调试时或者出错时用。
ps -aux|grep goproxy
kill xxxxx

测试：

y2b 720p 视频测试（单开）。下面是1分钟动图。

CPU 占用看起“很高”，但你算下，前面几个加起来都超过 100% 了。。。。
使用过程中并没有延时的感觉 ---- 不管是网页还是视频。
最后最重重要的，耗电问题 ---- 垃圾手机一小时一天能耗多少电 :)

（goproxy 部分完）
