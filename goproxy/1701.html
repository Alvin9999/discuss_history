之前在路由器上部署，一直出问题，昨天终于全部搞定。积累了几个要点。分享如下
1、路由器，要安装 mipsle架构的 goproxy版本，比如我 的机器安装的就是：
goproxy_linux_mipsle-r1389

要点：尽管按照路由器 uname -m的提示，需要安装mipse版本，但是实际安装后，一直提示错误。最后只有安装mipsle版本，才能无误。

2、先将goproxy_linux_mipsle-r1389版本下载到Mac本机里。然后利用scp将修改过gae.json 的这个版本上传到路由器的 data目录
scp -r ~/Downlodas/goproxy_linux_mipsle-r1389 root@XXXX.XXXX.X:/data 【你的路由器地址】

要点：有些路由器不支持sftp，但是大部分都支持scp命令。要用scp将下载的gp进行上传路由器目录

3、将路由器上的监听端口改为0.0.0.0.1:8087

cd /data/goproxy_linux_mipsle-r1389
vim httpproxy.json
修改这个文件里的端口即可

要点：必须更改，否则和路由器连接的设备无法使用gp

4、在路由器上第一次运行goproxy
cd /data/goproxy_linux_mipsle-r1389
./goproxy
第一次运行会耗费10来分钟安装证书【但是这个证书是不管用的】

5、随便将Mac机上，不管那个版本的goproxy的目录下的 cache 文件夹 整体上传到 路由器上的goproxy目录 【这一步非常重要，否则无法上YouTube等网站】
注意：
1】上传前，先将路由器上的 goproxy目录的cache文件夹整体删除 【只有执行过第4步，也就是在路由器上运行过一次goproxy，才能有这个文件夹】
2】上传前，如果Mac笔记本 的goproxy文件在Applications目录，则需要使用 sudo 进行scp 上传
以下2步对应 上述2项 注意事项
1】cd /data/goproxy_linux_mipsle-r1389
rm -r cache/
2】sudo scp -r /Applications/goproxy/cache root@XXXX.XXXX.X:/data/goproxy_linux_mipsle-r1389

6、运行过以上所有步骤后
第二次执行
./goproxy

 由此， 在路由器上的设置全部完成

7、在电脑端设置
找到电脑端网络设置，在代理选项中选择 自动，键入路由器上运行的 ./goproxy程序产生的 Pac Server，然后在chrome中选择 系统代理 即可

比如：http://20.200.21.85:8087/proxy.pac 【你的gp在路由器上运行后，产生的地址端口】

要点：必须在chrome的sharpomega中选择，系统代理，如果选择直接连接，是不会有反应的

8、在电脑端或移动设备端，安装CA证书
在电脑或移动设备端的浏览器上，键入20.200.21.85:8087【你的gp在路由器上运行后，产生的地址端口】，将其产生crt文件下载，安装到本机设备上

要点：必须在电脑本机或移动设备上，安装gp在路由器上产生的crt文件。否则无法访问youtube 等网站

因为不同路由固件的问题，我觉得较好的办法是利用jffs部署entware 或者 optware,获得完整的pyssl环境，然后很多问题都容易解决。
开机启动一般部署entware之后都会在jffs生成
/jffs/scripts
目录包含这个文件或者手动创建services-start，然后在这输入启动命令就可以了。
