嗯，只要不到 100% 都还好。100% CPU 占用比较头疼，目前在 mips 路由器上几乎必现。

尤其签署证书这一步。建议使用 mips 版的 goproxy 把 PC 上的证书拷过去然后再启动。

然后就静待 tls 1.3 了。

  1. chrome 的 tls1.3 已经可以通过 chrome://flags 开启
  2. golang 的 tls1.3 已经由 cloudflare 完成，https://github.com/cloudflare/tls-tris 我随时可以移植。
  3. google frontend server 不知道什么时候部署 tls1.3

所以只要第 3 布完成，goproxy 就可以第一时间用上 tls 1.3 了。

goproxy-vps r142 已经支持 tls1.3 了。在 chrome://flags/#ssl-version-max 可以打开 chrome 的 tls1.3 就能一起使用。

目前 goproxy-vps 的 tls1.3 是使用的 cloudflare 的补丁，经测试首次连接的效率提高比较明显。
