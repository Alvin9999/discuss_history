phuslu commented Feb 16, 2017 • edited

现在我比较关注 tls 1.3 在谷歌服务器的部署情况了。

  1. https://bugs.chromium.org/p/boringssl/issues/detail?id=76
  2. https://bugs.chromium.org/p/chromium/issues/detail?id=641225

这个问题解决以后，预计谷歌 GFE 服务器就可以全部上 tls 1.3，那样的 goproxy 的体验就能再上一个台阶了。

和 goproxy http2 这个依赖比较好的网络状况的特性不一样。goproxy 的 tls 1.3 的 1-RTT 和 0-RTT 恰恰适合现在的干扰剧烈的恶劣网络了。

phuslu commented Feb 17, 2017 • edited

r1297 881bf3d 打开了浏览器和客户端之间的 tls1.3 支持。

虽然这对速度提升不大，但是可以有效的降低一些电脑的 CPU 占用率了。

如上图所示，我的笔记本 CPU 是比较旧的 T9200， 不支持 AES-NI 只能. 使用了 tls1.3 之后的 x25519/chacha20 可以降低 CPU 使用率了。 😄
