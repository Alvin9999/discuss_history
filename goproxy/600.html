MeABc commented Apr 26, 2016

@phuslu

对状态位的解释：还是这里 5862f66

就是增加一个类似这样的（IP池？）：
IPn = tr

// n = 1-->10 或 n = 1-->n

前言：
社长也说了，这种探测因为塞到了 TLS 握手包里面了， 不会存在任何开销。
但我们的目标不是探测，而是识别。找了一下，是可以识别的。目标是想保存IP(服务器)的h2值。
参考如下：
http://www.kancloud.cn/digest/web-performance-http2/74828
http://www.blogjava.net/yongboy/archive/2015/03/18/423570.html

解释：
第一轮ip测试和识别下来，就可以 IPn = tr
简单说就是让程序自动处理 "DisableHTTP2" 和 "ForceHTTP2" 这两部分。
现在实现上已扩展变成，状态位(标志位 Flag) + IP池了。

目标：
a. 让社长不要再纠结 "DisableHTTP2" 和 "ForceHTTP2" 默认值是什么。
b. 最终去掉这两个配置项。程序已自动处理。
c. 在此基础上，可以定时处理IP池的好坏(如 IP池缓存10个，填20个IP，IP池如有坏的就补入)
d. 未来更平滑的支持 googlezip IP。只须一个标志位就可以调用相应模块。

好外：
a. 体验更平滑。(goproxy有一现象，隔久不用会卡住，须重载。)

又一个现象：有时使用过程中会提示badIP，不能FQ，但重载就好，不是IP问题，个人认为 "标志位 + IP池" 可以解决这个问题。
